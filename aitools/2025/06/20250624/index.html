AI新工具(20250624) 基于MCP的智能学术研究助手ScholAI；Claude Code OpenAI兼容代理

✨ 1: c4 GenAI Suite

生成式AI聊天套件

c4 GenAI Suite是一个高度可配置且可扩展的AI聊天机器人应用，其核心优势在于通过Langchain深度整合模型上下文协议（MCP），并广泛支持OpenAI、Azure OpenAI、Bedrock、Google GenAI、Ollama等主流大语言模型及嵌入模型。该平台允许管理员创建并定制化AI助手，通过集成RAG（检索增强生成）服务或各类扩展来赋予助手多样化能力。尤其值得关注的是，项目内置了强大的REI-S（检索、提取、摄取服务器）模块，该模块能够处理各种文件格式（包括PDF、Office文档及通过Whisper进行语音转录的音频），实现高效的文件内容提取、索引和基于pgvector或Azure AI Search等向量存储的检索增强功能，从而使助手能针对领域特定问题提供精确的上下文感知回答。整个系统以React前端、NestJS后端和Python FastAPI的REI-S服务构建，强调模块化设计，确保了用户交互的流畅性、助手功能的丰富性及平台整体的灵活性。

地址：https://github.com/codecentric/c4-genai-suite

✨ 2: ScholAI

基于MCP的智能学术研究助手

ScholAI是一个基于模型上下文协议（MCP）的智能学术研究助手，致力于提升学术研究工作效率，其核心功能涵盖了多维度学术出版物检索，包括从arXiv预印本和专业会议期刊中发现相关论文，并能自动获取其CCF排名。该项目还具备强大的PDF文档管理能力，允许用户下载并高效提取论文文本内容。尤其值得注意的是，ScholAI提供语义查询分析，可以将用户的自然语言研究兴趣逐步细化并转化为精确的学术查询，从而全面助力学术发现、分析与管理。

地址：https://github.com/oDaiSuno/ScholAI

✨ 3: Claude Code Proxy

Claude Code OpenAI兼容代理

Claude Code Proxy是一个至关重要的代理服务器，其核心功能是实现Claude Code命令行工具与任何OpenAI兼容的API提供商的无缝对接。该项目通过实时转换Claude API请求（尤其是/v1/messages接口）为OpenAI API调用来达成这一目标，从而使用户能够灵活利用包括OpenAI、Azure OpenAI、本地Ollama模型以及其他通用OpenAI兼容服务。其关键特性包括智能模型映射，可根据Claude模型的类型（如“haiku”、“sonnet”或“opus”）自动路由到用户配置的“大模型”或“小模型”；此外，它还提供了全面的函数调用（工具使用）支持、实时流式响应能力以及对Base64编码图像输入的处理，所有这些都通过简洁的环境变量配置，确保了高度的灵活性和便捷的集成体验。

地址：https://github.com/fuergaosi233/claude-code-proxy

✨ 4: SnapAI

AI 应用图标生成器

SnapAI是一款利用OpenAI图像生成模型为React Native和Expo开发者快速创建专业应用图标的工具，其主要功能和核心亮点在于：支持通过简洁的文本提示词迅速生成高清、多尺寸（包括iOS优化）的图标，且操作界面为开发者友好的命令行接口，便于集成至CI/CD流程实现自动化。该工具高度重视用户隐私，API密钥本地存储，不收集任何用户数据，且自身免费使用，用户仅需支付OpenAI的生成成本。

地址：https://github.com/betomoedano/snapai

✨ 5: Dual AI Chat

双AI智能辩论问答

Dual AI Chat项目是一款创新的Web应用，其核心功能是驱动逻辑型AI (Cognito) 和怀疑型AI (Muse) 之间进行内部辩论，旨在通过这种协作过程生成更健全、准确且经过严格审查的响应，有效减少AI幻觉。该项目支持灵活的AI后端，默认集成Google Gemini API，并兼容任何OpenAI API（包括本地模型如Ollama），用户可上传文本和图片进行多模态输入。其关键特性还包括一个带有撤销/重做功能的共享记事本，作为AI协作空间并保持对话状态，以及高度可配置的设置，允许切换AI模型、配置API参数和选择对话模式（固定轮次或AI自主驱动）。此外，系统具备强大的错误处理机制，支持API调用的自动重试及手动恢复，确保了对话的连续性，最终由Cognito在记事本中呈现综合性的答案。

地址：https://github.com/yeahhe365/Dual-AI-Chat
